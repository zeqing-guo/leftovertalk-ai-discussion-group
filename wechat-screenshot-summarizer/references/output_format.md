# 截图总结格式指南

本指南用于指导 agent 从微信群聊截图中提取**工具推荐**和 **AI 使用经验分享**，并按统一格式输出总结。

## 总结流程

1. 读取截图图片，识别所有发言内容
2. 提取每条发言的**发言人姓名/昵称**和**具体内容**
3. 重点识别：推荐的工具/产品、AI 使用技巧和经验分享
4. 对信息不完整的推荐（例如只提了名字没给网址），**必须使用 search 工具查找并补全产品名称、用途和官网地址**
5. 按下方格式输出 markdown 总结文件

## 输出文件命名

输出文件名与截图文件名对应，格式为 `图片名字.md`。

例如：截图文件为 `01-13-Recorder_Scroll_2026-02-02_16-41-59.png`，则输出文件为 `01-13-Recorder_Scroll_2026-02-02_16-41-59.md`。

## 输出格式模板

```markdown
# [截图文件名]

## 工具推荐

### 1. [工具/产品名称]

- **推荐人**：[发言人姓名]
- **时间**：[从截图中提取的讨论时间，如 2026-01-13 14:30]
- **用途**：[工具的功能和使用场景]
- **网址**：[官方网址]
- **原图来源**：![截图](./截图文件名/part_NNN.png)（如果图片被分割，链接到内容所在的子图片；如果图片未分割则链接原图：![截图](./截图文件名.png)）

### 2. [工具/产品名称]

- **推荐人**：[发言人姓名]
- **时间**：[从截图中提取的讨论时间]
- **用途**：[工具的功能和使用场景]
- **网址**：[官方网址]
- **原图来源**：![截图](./截图文件名/part_NNN.png)（同上规则）

（如本张截图无工具推荐，此部分写"无"）

## AI 使用经验分享

### 1. [经验主题/关键词]

- **分享人**：[发言人姓名]
- **时间**：[从截图中提取的讨论时间，如 2026-01-13 14:30]
- **经验**：[分享的经验、技巧或心得，忠实还原原文意思]
- **原图来源**：![截图](./截图文件名/part_NNN.png)（如果图片被分割，链接到内容所在的子图片；如果图片未分割则链接原图：![截图](./截图文件名.png)）

### 2. [经验主题/关键词]

- **分享人**：[发言人姓名]
- **时间**：[从截图中提取的讨论时间]
- **经验**：[分享的经验、技巧或心得]
- **原图来源**：![截图](./截图文件名/part_NNN.png)（同上规则）

（如本张截图无经验分享，此部分写"无"）
```

## 格式要求

### 发言人

- **必须写出发言人的具体姓名或昵称**，禁止使用"群友""有人""某人"等模糊称呼
- 如果截图中能看到头像和昵称，以昵称为准
- 如果同一个人多次发言，每条都要标注姓名

### 工具推荐

- **工具名称**：必须写明具体的产品/工具名称
- **时间**：必须从截图中提取该条讨论发生的时间（微信消息上方的时间戳），格式如 `2026-01-13 14:30`
- **用途**：必须说明该工具是做什么的、解决什么问题
- **网址**：必须提供官方网址或可访问的链接
- 如果发言人提到的信息不够完整（例如只说了工具名没说网址，或者只描述了功能没说名字），**必须使用 search 工具查找并补全信息**
- 每一条推荐都必须附上原图来源，方便溯源。如果图片被分割为子图片，原图来源应链接到内容所在的具体子图片（如 `./截图文件名/part_001.png`），而非原始长图

### AI 使用经验分享

- 使用与工具推荐相同的编号格式（### 1, ### 2, ...）
- **分享人**：必须写出发言人的具体姓名或昵称
- **时间**：必须从截图中提取该条讨论发生的时间，格式同工具推荐
- **经验**：忠实还原发言人的原意，记录具体的使用技巧、心得或建议；如涉及具体的 prompt、工作流或操作步骤，尽量完整记录
- **原图来源**：与工具推荐相同规则，链接到内容所在的子图片或原图

### 其他注意事项

- 使用中文撰写总结
- 保持客观，忠实还原发言内容，不添加主观评价
- 只提取工具推荐和 AI 使用经验相关内容，忽略闲聊、表情包等无关信息
